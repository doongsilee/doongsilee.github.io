# R을 활용한 대용량 데이터 처리

R은 대용량 데이터 처리에 특화되어 있는 프로그래밍 언어다.
R을 활용하면 이러저러한 많은 일들을 처리할수 있다.
라고 한다. 근데 정확히 뭘하지? 이글은 평범한 클라이언트,서버 개발 경험만 했던 개발자가 맨땅에 R딩하기에 관련된 글이다. 미흡하게나마
이 글을 통해 나처럼 R을 처음 접하는 다른분들에게도 도움이 되길 바란다.

우선 갑자기 R에 대해 고민을 시작하게 된 계기부터 얘기해본다. 왜 R을 선택했는가도 나오기때문에 서론이지만 나름 중요하다

그러니까 이 고민이 시작된건 불과 몇일전에 평화롭던 어느날, 오늘도 안드로이드 개발에 여념이 없던 온순한 개발자에게 어마어마한 일이 떨어진다.
일인 즉슨 우리 서비스의 추천 및 검색 정확도를 올리기위해서는 데이터가 많아야하는데
다행이 이번에 많은 데이터를 파일 형태로 받아왔으니 이걸 잘 정제해서 디비에 넣어야한다는 것이었다.

언뜻 듣기에 별것 아닌것 같아보였기에(예전회사에서 어드민 포탈 만들때 대충 csv 파싱하는 모듈을 만들어보기도 했었고)
제가 하죠 뭐 라고 대답해버렸다. ~~그때 난 그러지 말았어야했다.~~

조금 더 디테일 한 요구사항을 뽑아보자.

1. 신규로 받아온 대용량 CSV 파일과 대용량의 Excel 파일들이 있다.
   이 두가지 파일들은 서로 다른곳에서 가져왔기에 나타내는 데이터는 비슷하지만 포맷은 상당히 다르다.
   여기서 우리에게 필요한 데이터들을 추출한다.
2. 추출한 데이터중에 중복된 데이터들을 삭제한다.
3. 이 데이터들을 데이터베이스에 저장한다. 이때 기 저장되있는 데이터와 신규 추가되는 데이터 사이에도 중복이 발생할 수 있다. 이때는 저장하지 않고 업데이트를 쳐야한다.
4. 이 작업은 일회성이 아니고 분기(혹은 반기) 별로 신규 데이터 소스가 추가될때 다시 수행되어야하고 이때마다 데이터 포맷이 조금씩 달라질 수도 있다.

위에 요구사항을 충족하는 해결책은 뭐가 있을까

처음 든 생각은 우리가 기존에 사용하던 그대로 Java 를 사용하는 방법이었다.
기존 우리의 서비스로직이 Spring, mybatis 등으로 구현되었기에 Java로 stand alone Application을 만들면 기존 코드도 재활용 할수 있으리라.
문제는 우리가 가지고 있는 데이터들의 사이즈였다.
csv 파일 포맷으로 된 데이터가 약 200만건 , excel에 약 300만건 , database 에 기 저장되있는 데이터도 약 200만건을 상회했다.
자바로 불가능할것도 없어보이긴 했지만 이 모든 데이터들을 끊임없이 파싱하고 비교하면서 중복검사를 수행하는 일이 과연 자바로 하기에 적합한 일인지 의문을 가질수 밖에 없었다.

두번째는 Python을 활용한 방법이다.
현재 팀에서는 Java를 주로 활용하고 있지만, 개인적으로는 Python을 더 좋아하기도 했고, 팀에서도 현재 우리가 따로 만들지 않은
Admin Site만이라도 향후에 Django Admin을 활용해서 간단하게 구축하려는 계획도 가지고 있었기에 사용자체는 큰 문제는 없어보였다.
하지만 자바와 마찬가지로 여러줄의 데이터를 가져와서 클렌징하고 끊임없이 비교하고 이를 데이터베이스에 SQLAlchemy 같은 ORM 솔루션
적용해서 넣는 작업이 그다지 적합하다는 느낌은 들지 않았다.(아 물론 data Wrangling에 관련된 다양한 python 솔루션도 많이 있긴 하더라)

마지막으로 searching을 통해 찾아낸 것이 바로 R 이다.
사실 R은 많이 들어보긴 했지만, 내게 데이터를 분석하고, 그래프를 그리는 따위의 일을 한다고 대충 알고만 있었는데
찾아볼수록 현재의 작업에 어울리는 언어라는 생각이 들었다.
R은 다양한 소스에서부터 데이터를 가져와 이를 클렌징하고, 통합하고 데이터베이스에 저장한다.
추가로 이번기회를 통해 R을 습득해두면 향후에 언젠가 해야할 데이터 분석에 있어서도 큰 도움을 받을수 있을것 같았다.
