### 어떤 암호화 방법을 택할까?

자 시작해보자. 늘 그렇듯이 아는게 없으므로 맨땅에 헤딩하면서 시작이다.
DB를 암호화 해야한다. 어떻게 하지? 당장 드는 생각은
"음 서버 소스코드를 수정해서 저장할때 암호화(encrypt) 해서 넣으면 되나?"
나쁘지 않아 보인다. 가져올때는 암호화되지 않은 key 를 가지고 select 해서 가져와서 decrypt 하면 되겠지 싶다.
잠깐만.. 이렇게 간단한가? 여기서 문제는?

일단 소스코드 수정은 매우 귀찮은 작업이다. ~~진짜 죤나 하기 싫다~~.
암호화할 라이브러리도 찾아야 되고 사용방법도 확인해야하고 이래저리 시간이 꽤 걸릴듯 하다.
두번째 데이터 쿼리문 where절에 암호화된 정보가 들어가야한다면 문제가 생길듯하다.
생각해보자 만약 item 을 그냥 찾으면 되는 where 문이라면 Server 에서 encrypt 한 결과로 쿼리를 작성하면 어떨까? 예컨데 이런식으로

```SQL
SELECT * from encrypted_table where item=asdf123145(encrypt 된 값)
```

놀랍게도 가능하단다. [여기](stackoverflow.com)참고.
그런데 만약 이런 경우는 어떡하지?

```SQL
SELECT * from encrypted_table where item between a23s34 from a234df1
```

아이런 where 절에 범위 조건같은게 들어가야한다면 문제다. 암호화된 값들 사이에 어찌 제대로 범위를 찾을수 있겠는가. 그리고 안타깝게도 우리 서비스는 이 쿼리가 필요하다. ~~갓뎀~~

찾아보니 이부분에 대해서는 이미 많은 사람들이 고민을 했었다.

데이터베이스 암호화의 방법은 크게 필요한 Column 만 암호화하거나 테이블 or database를 전체 암호화 하는 방법 이렇게 2가지로 나누는 경우도 있고
아니면 우리같은 개발자 입장을 배려하여 소스코드를 바꿔야 되거나 아니면 안바꿔도 되거나 2가지 방법으로 나눌수도 있겠다.
