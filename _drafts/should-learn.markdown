## 웹 클라이언트 개발 계획

개발을 본격적으로 시작한게 군대 전역하고 소프트웨어멤버십에서 활동을 시작한 2009년이라고 보고
어느덧 개발을 시작한지 햇수로는 거의 10년이 다 되간다.
그 동안 나름대로 다양한 분야에 발을 담궈왔다.

시작은 나를 개발자의 길에 들게 만들어준 아이폰(iOS) Platform 부터였다. 이후 Android 를 거쳐 지금은 없어진 삼성의 'bada' 까지 모바일 프로그램 개발을 주로 담당했다.
삼성 인프라팀에서 일할땐 AWS 같은 Cloud Infra 와 Backend 기술들(주로는 운영이나 셋업에 치중됫지만) 그리고 무엇보다 Linux에 대한 소중한 경험을 얻었다.

회사를 창업하고나서도 새로운 경험을 했다. 백앤드가 필요하기에 Django 를 써서 백앤드를 구성했고
이걸 다시 Spring boot으로 변경하기도 했다.
가장 생뚱맞았던 분야는 R이다. 프로젝트를 진행하다보니 데이터 클렌징이 필요했고 관련 제반지식이 전혀 없는 와중에 R을 활용해보기도 했다.

이 얘기들이 나 이거저거 해봤소 라고 자랑하려는건 아니다. 이렇게 이것저것 (비록 깊이는 깊지 않다해도) 손을 대본 와중에
내가 거의 손 대지 못했고 가장 자신 없는 분야가 바로 웹 클라이언트라는 말을 하고 싶었다.

물론 웹이라고 아예 손을 댈 기회가 없었던건 아니다. ~~기본적으로 HTML,CSS 는 다른 개발을 해도 필요했으므로 알고 있다.~~
(최근 다시 보다보니 깨달았는데 난 정말 최소한의 기초적인 지식만 있었다.)
그 외에도 주로 어드민(백오피스) 프로젝트를 하며 Bootstrap , Angular 를 살짝 맛보기도 했었고
개인적으로 블로그를 만들기 위해서 wordpress 나 jekyll 같은 라이브러리들을 보기도 했었다.(지금도 글을 쓰고 있는 이 블로그도 jekyll로 만들어져있다!!)

그럼에도 불구하고 여전히 난 웹 클라이언트에 대해 거의 모른다.
여전히 Javascript code 는 보기에 괴의하고 이해할수가 없으며(어이 없는건 Javascript 공식 레퍼런스나 책을 훓어보면 Javascript는 어렵지 않다. 변수니 함수니 제어문이니 전부 다 내가 알던 언어들과 그다지 다르지도 않은데 왜 코드만 보면 이리도 무서운가??), 오만가지 다양한 sass 니 scss 니 하는 끔찍한 혼종들은 css와 뭐가 다른건지 어디서부터 봐야할지 모르겠다. webpack이니 babel 이니 하는 **기본적**이라는 툴들은 또 어떤가? 각각의 도큐먼트들만 읽어보려해도 한 세월일 것 같아 엄두가 나질 않는다.
거기에 새로운 기술들 나오는 속도는 또 어떤지 생각해보라.
작년 Angular를 한번 봐볼까 했을때 버전이 2.0 이었던거같은데 얼마전에 지나가다보니 6.0 ?
요즘은 다들 Next로 개발합니다. 그게 뭐죠? React니 vue니 그런애들도 모르는데 그런애들은 또 한번 감쌋다고요?
서버사이드 렌더링이요? 요즘엔 CSS in JS 컨셉으로 개발된 라이브러리들이 많이 나온다고요?

(하..)

이러한 이유들로 난 번번히 웹 클라이언트 개발 공부를 했고, 업무상 필요에 의해 그때그때 어찌어찌 뭔가를 만들어내기도 했지만
웹 클라이언트 개발을 모른다. 전혀.

이제는 달라져야할것 같다. 블록체인을 봐도 주로 클라이언트는 웹이며, 각종 모바일 앱도 React Native 등으로 생산성 있게
개발하는 것이 가능하고(나중에 다시 네이티브로 가는 경우도 많다는건 알고 있다) 서버도 Node를 이용해 만드는 이 분위기에 자바스크립트 그리고 웹 클라이언트를 모른다는건 개발자로써 개발의 매우 큰 축을 모른다는 생각이 든다.
더구나 신규로 시작하는 사업은 웹 기반으로 진행해야하기에 난 더이상 물러날곳 없이 다시한번 웹 클라이언트라는 괴물 앞에 서게 됬다.
지금까지랑은 다르게 이 프로젝트 이후로는 전 웹 개발을 할줄 아는 개발자입니다 라고 말을 하려면
계획부터 철저하게 세워 시작하는게 좋겠다.
